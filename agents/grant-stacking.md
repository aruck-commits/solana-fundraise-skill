# Grant Stacking Agent

A dedicated agent that builds you a multi-program funding strategy — so you're applying to the right programs in the right order and maximizing your chances of getting funded.

## What it does

Most founders apply to one grant at a time and wait. That's slow. The right move is to stack — apply to multiple programs in sequence, using each win to build credibility for the next.

This agent builds your personal stacking strategy based on your project and current stage.

---

## How to use it

Tell Claude Code:
> "Build me a grant stacking strategy for my project"

Then describe:
- What you're building
- Your current stage (idea / prototype / live)
- Your region
- Any prior grants or bounties won
- Your timeline (when do you need funding by?)

---

## The stacking playbook

### For founders at idea stage

```
Week 1-4:   Win 2-3 Superteam Earn bounties
            → Builds profile, gets you known in the community

Week 4-8:   Apply for Superteam Meetup Fund or small regional grant
            → First grant, low bar, builds track record

Week 8-16:  Build prototype using early funding
            → Now you have something to show

Week 16+:   Apply to Solana Foundation with prototype + grant history
            → Foundation application is now credible
```

### For founders with a prototype

```
Immediately: Apply to Superteam grant (fast turnaround, 1-3 weeks)
             → Use the time to prep Foundation application

Week 2-4:    Submit Solana Foundation application
             → Superteam grant in review gives you social proof

Week 4-8:    Enter next Colosseum hackathon
             → Hackathon placement adds third credibility signal

Result:      3 programs in pipeline simultaneously
```

### For founders with a live product

```
Immediately: Apply to Solana Foundation (strongest application now)
             → Live product = lowest risk for reviewers

Simultaneously: Enter Colosseum hackathon if timing works
               → Hackathon win can unlock accelerator

After Foundation: Use grant to build toward accelerator metrics
                  → Colosseum accelerator = $250K pre-seed
```

---

## The stacking rules

**Rule 1: Superteam first, always**
Superteam is fastest (1-3 weeks), lowest bar, and adds credibility to every subsequent application.

**Rule 2: Never wait for one decision before applying to the next**
Applications can be in flight simultaneously. Decisions take weeks — use that time.

**Rule 3: Each win funds the next application**
Use Superteam grant to build the prototype that gets you the Foundation grant. Use Foundation grant to build the product that gets you into Colosseum accelerator.

**Rule 4: Document everything publicly**
Every milestone, every ship, every user — put it on GitHub and X. Grant reviewers Google you.

**Rule 5: Reject → Improve → Reapply**
A rejection from Superteam means fix the application and reapply in 4-6 weeks with more proof of work. Don't move on — fix it.
