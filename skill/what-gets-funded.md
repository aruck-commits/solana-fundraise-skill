# What Actually Gets Funded

Patterns from funded Solana projects across Foundation, Superteam, and Colosseum. If you're still deciding what to build — or how to frame what you're already building — start here.

---

## What the Solana Foundation has funded (patterns)

The Foundation funds public goods. Here's what that looks like in practice:

**Developer tooling that removes real friction**
- Local development environment improvements
- Testing frameworks (LiteSVM, Mollusk integrations)
- CLI tools that automate repetitive setup
- Error diagnosis tools (the kind that save hours when things break)

**Infrastructure with ecosystem-wide benefit**
- Indexers and RPC improvements
- On-chain data tooling
- Explorer features
- Cross-program composability tools

**Education that's actually deep**
- Technical documentation for underserved Solana primitives
- Developer onboarding resources with working code examples
- Migration guides (e.g. Anchor version upgrades, web3.js to Kit migrations)

**Security**
- Audit tooling
- Formal verification research
- Vulnerability databases and detection tools

**Pattern:** Foundation-funded projects are things that a solo developer would never build because there's no business model — but the whole ecosystem suffers without them.

## What Superteam has funded (patterns)

Superteam funds people and regional impact. Here's what works:

**Community infrastructure**
- Telegram bots for regional Superteam chapters
- Bounty tracking tools for Superteam Earn
- Notification systems for ecosystem events
- Translation of technical content into regional languages

**Regional education**
- Developer workshops in emerging markets
- Content explaining Solana concepts in local context
- IRL event series (meetup grants are small but fast)

**Early-stage products with a regional angle**
- Consumer apps addressing problems specific to a region
- Payment tools for markets underserved by crypto infrastructure
- Creator economy tools for regional communities

**Content with ecosystem value**
- Deep-dive threads on underexplored Solana topics
- Explainer content that simplifies complex technical topics
- Video guides for new Solana developers in specific regions

**Pattern:** Superteam funds things that make the Solana ecosystem stronger in a specific geography, or things that help Superteam members do their work better.

## What Colosseum has funded (patterns)

Colosseum funds startups. The question isn't "is this a public good" — it's "could this be a real company."

**Recent winning categories (Frontier 2026 and prior hackathons):**
- AI agents with on-chain capabilities
- DePIN infrastructure with real hardware integrations
- Consumer apps with novel token mechanics
- Cross-chain infrastructure solving real liquidity problems
- Developer tooling with a clear monetization path

**What Colosseum judges reward:**
- Working demo over perfect pitch
- Novel primitive over incremental improvement
- Clear market thesis over vague "web3 is the future" framing
- Team that looks like they'll keep building over solo founders with no track record

**Pattern:** Colosseum winners solve hard problems with working code and can explain in one sentence why their startup has a real market.

## What doesn't get funded (across all programs)

**Anything that's already been funded**
If a similar project received a grant in the last 12 months, your application needs to explicitly differentiate. "We're doing X but better" is not differentiation.

**Pure token projects**
Projects where the token is the product. Grant programs fund infrastructure and tools, not launches.

**Marketing-only projects**
"We'll tweet about Solana and run a Discord" doesn't get funded without a technical component or significant existing community.

**Vague AI projects**
"AI + blockchain" with no specific use case, no working code, and no clear user. The space is flooded with these.

**Projects the founder doesn't use**
Grant reviewers can tell when a founder isn't solving their own problem. The best applications come from builders who are frustrated with a real gap they personally hit.

## The highest-probability funding paths in 2026

Based on what's actually getting approved right now:

**Highest probability:**
1. Dev tooling that removes specific friction (Foundation)
2. Regional community infrastructure (Superteam)
3. AI agents with real on-chain capabilities (Colosseum)

**Medium probability:**
1. Consumer apps with clear regional angle (Superteam)
2. Security tooling (Foundation)
3. DePIN with working hardware (Colosseum)

**Lower probability:**
1. Pure content projects without a product component
2. "We're building X" with no working prototype
3. Anything targeting a problem already solved by a funded team

## The fastest path from idea to funded in 2026

1. Build something you personally use and are frustrated with
2. Check if it already exists — if it does, find a specific gap
3. Win 2-3 Superteam bounties to build profile
4. Ship a rough prototype and put it on GitHub
5. Post about the problem on X — see if anyone resonates
6. Apply to Superteam first (fastest decision)
7. Use Superteam grant to build toward Foundation-quality product
8. Apply to Foundation with proof of delivery from Superteam milestone

Total timeline from idea to Foundation-funded: 3-6 months if you move fast.
